1) The Problem

Iranian creators are cut off from global monetization platforms due to sanctions and payment restrictions. Patreon, OnlyFans, Whop — none work. This forces them to either rely on unstable workarounds or give up on monetization entirely.

The gap is clear: independent creators, educators, artists, and influencers need a reliable local alternative to build sustainable income without depending on shaky third-party solutions.

2) The Solution

Yarplus is a creator monetization platform built specifically for the Iranian market. It allows creators to set subscription tiers, manage exclusive content, and process payments through local gateways — all in one place.

Core features include: creator dashboard with analytics, subscriber management, tiered access control, automated billing, and a discovery feed for fans. The backend uses a microservices architecture to keep things scalable and maintainable.

Next.js Node.js PostgreSQL TypeScript Tailwind CSS

3) Key Learnings

This project meant tackling payment processing in a restricted environment while keeping security and user experience tight. Handling sensitive financial data required careful thought around privacy, fraud prevention, and local regulations.

I used AI-assisted development to speed things up — letting me focus on architecture, business logic, and UX instead of getting stuck in boilerplate. It showed how the right tooling can boost velocity without losing control over the product.
